1. Capítulo 01 - Introdução

Escrito por Uianes Luiz Rockenbach Biondo.
Última atualização em 15 dez. 2024

A crise dos desenvolvedores de Software

Anos 1970:

  1. Desenvolvedores utilizavam processos não adequados.
  2. Os processos adequados não eram empregados por falta de tempo e recursos.
  3. Dijkstra (1971) foi o primeiro a utilizar o termo "crise do software".
  4. Os problemas elencados por Dijkstra foram:
    1. Cronograma;
    2. Orçamento estourado;
    3. Baixa qualidade, ineficiência e não atendimento de todos os requisitos no produto final;
    4. Produtos não gerenciáveis, como difícil manutenção e evolução;
    5. Produtos nunca entregues;
  5. Wazlawick aponta que a crise do software continuará enquanto forem utilizados processos artesanais e não forem capitalizados os erros e acertos.

Os eternos mitos:

São identificados inicialmente por Pressman (2005). São agrupados em mitos:

  1. Administrativos;
  2. Do cliente;
  3. Do profissional;
  1. Mitos Administrativos
    1. A existência de um manual de procedimentos e padrões é suficiente para a equipe produzir com qualidade;
    2. A empresa deve produzir com qualidade, pois tem ferramentas e computadores de última geração;
    3. Se o projeto estiver atrasado, sempre é possível adicionar mais programadores para cumprir o cronograma (Lei de Brooks);
    4. Um bom gerente pode gerenciar qualquer projeto;
  2. Mitos do Cliente:
    1. Uma declaração de objetivos é suficiente para iniciar a fase de programação;
    2. Os requisitos mudam com frequência, mas sempre é possível acomodá-los, pois o software é flexível;
    3. Eu sei do que preciso;
  3. Mitos do profissional
    1. Assim que o programa for colocado em operação, nosso trabalho terminou;
    2. Enquanto o programa não estiver funcionando, não será possível avaliar sua qualidade;
    3. Se eu esquecer alguma coisa, posso arrumar depois;
    4. A única entrega importante em um projeto é o software funcionando;

Leverson (1995) apresenta outros mitos, acerca da confiabilidade de software:

  1. O teste do software ou sua verificação formal pode remover todos os erros;
  2. Aumentar a confiabilidade do software aumenta a segurança;
  3. O reúso de software aumenta a segurança;

(In)Definição da Engenharia de Software

Engenharia de Software: "O processo de estudar, criar e otimizar os processos de trabalho para os desenvolvedores de software" (WAZLAWICK, 2019, p. 4)

O Engenheiro de Software

Conceitos como "desenvolvedor" e "engenheiro de software" geram confusão.

Desenvolvedor: Profissional que executa o projeto.

Engenheiro de Software: Profissional que cria condições para o trabalho do desenvolvedor. Escolhe e especifica o processo de planejamento para todos os projetos.

Outros cargos ou funções:

Gerente de projeto: Cuida de um projeto específico (prazos, orçamento, escopo e objetivos);

Analista: Desenvolvedor que realiza o levantamento de requisitos e modelagem;

Designer: Define as tecnologias utilizadas para implementação dos modelos/requisitos definidos pelo Analista;

Programador: Cria a implementação física do projeto.

Evolução da Engenharia de Software

  1. Computadores entre 1930 e 1940 utilizavam programação física, sem software embutido, a programação era realizada diretamente no Hardware.
  2. A necessidade de criação do software fez surgir a engenharia de software.
  3. O conceito inicial era ligado a ideia de campo de conhecimento que produz o software.
  4. A primeira conferência de Engenharia de Software foi a Conferência de Engenharia de Software da OTAN, em Garmish (Alemanha) - 1968, mas o termo "Engenharia de Software" é utilizado desde 1950.
  5. O período entre 1960 e 1980 é compreendido como o "período da crise do software".
  6. Há um esforço contínuo para a solução da crise do software, inclusive nos dias de hoje, como a criação de: Ferramentas CASE (Computer Aided Software Engineering), especificação formal, prototipação, processos, componentes, testes de automatização, etc.
  7. Anos 90 consolidaram o WEB e a mudança de paradigmas para a orientação a objetos. Há uma maior preocupação com questões de segurança.
  8. Anos 2000: Consolidação dos métodos ágeis.
  9. Hoje: Busca de padronização (SWEBOK, ISO/IEC TR 19759).

Tipos de software do ponto de vista da engenharia

Software básico: compiladores, drivers, componentes do sistema operacional;

Software de tempo real: monitoram, analisam e controlam eventos do mundo real;

Software comercial: controles de estoque, vendas, etc. Geralmente aplicados em empresas. Chamados de Sistemas de Informações.

Software científico e de engenharia: voltados para o processamento de grande volume de dados.

Software embutido ou embarcado: presente em smartphones, eletrodomésticos, veículos, etc. Trabalham em um contexto de recursos computacionais escassos.

Software pessoal: softwares do dia a dia, como processadores de texto, planilhas (software de escritório). Genericamente são chamados de aplicativos.

Jogos: possuem especifidades que o distinguem do software pessoal.

Inteligência Artificial (IA): sistemas independentes, com processos de redes neurais ou formas de aprendizado.
